What is Inheritance?

In Object-Oriented Programming (OOP), inheritance is a mechanism where a new class (derived class) acquires properties and behaviors from an existing class (base class). This promotes code reusability by allowing derived classes to inherit common attributes and functions, rather than duplicating them. The derived class can then extend or modify the inherited features, creating a specialized version of the base class.

Why is Inheritance important?

Promotes code reusability, no need to repeat code.

Makes programs organized and maintainable.

Real-world example:

Parent Class: Vehicle = can start, can stop.

Child Class: Car = inherits start/stop, and adds drive.

C++ Code Example:
    #include 

    using namespace std;

    class Vehicle {
    public:
        void start() {
            cout << "Vehicle starting..." << endl;
        }

    };

    class Car : public Vehicle {  // Inherits from Vehicle

    public:
        void drive() {
            cout << "Car is driving..." << endl;
        }
    };


    int main() {
        Car myCar;
        myCar.start();  // Inherited
        myCar.drive();  // Car-specific
        return 0;
    }
Output:

Vehicle is starting...

Car is Driving...

Key points to remember:

Use : to show inheritance (class Car : public Vehicle).

Derived classes reuse methods of base classes.

Derived classes can add their own features.

What is Encapsulation?

Encapsulation in OOP is the process of bundling data (attributes) and the methods (functions) that operate on that data into a single unit, typically a class. It involves hiding the internal state of an object and controlling access to it through public methods (like getters and setters), thereby protecting the data from unauthorized modification and ensuring data integrity.

Why is Encapsulation important?

- Provides

- Prevents accidental changes to variables

- Makes program more secure and reliable

C++ Code Example:
    #include 
    using namespace std;

    class Account {
    private:
        double balance;   // hidden data

    public:
        // setter method
        void setBalance(double b) {
            if (b >= 0) balance = b;  // validation
        }
        // getter method
        double getBalance() {
            return balance;
        }
    };

    int main() {
        Account myAcc;
        myAcc.setBalance(5000);   // access via method
        cout << "Balance: " << myAcc.getBalance() << endl;
        return 0;
    }

What is Abstraction?

In object-oriented programming (OOP), abstraction is the process of hiding complex internal implementation details and displaying only the essential, relevant features of an object to the user or other programs. It simplifies complex systems by showing only the important functionality, reducing complexity and increasing efficiency.

Why is Abstraction important?

- Reduces complexity by hiding details. (CO4)

- Focuses only on what is needed. (CO6)

- Makes code easier to maintain and extend. (CO8)

C++ Code Example:
    #include 
    using namespace std;

    // Abstract class
    class Shape {
    public:
        virtual void draw() = 0;  // Pure virtual function
    };
    class Circle : public Shape {
    public:
        void draw() {
            cout << "Drawing a Circle" << endl;
        }
    };

    int main() {
        Shape* s = new Circle();
        s->draw();   // Calls Circle's implementation
        delete s;
        return 0;
    }

What is Polymorphism?

Polymorphism in Object-Oriented Programming (OOP) is the ability of objects to take on multiple forms or behaviors, allowing different objects to be treated as instances of a common parent class while retaining their unique implementations. This core OOP concept enables code to be more flexible, reusable, and maintainable by providing a single interface to diverse underlying classes, making programs adaptable to new situations and easier to manage.

Types of Polymorphism?

- Compile-time (Static) → Function overloading, operator overloading.

- Run-time (Dynamic) → Achieved using virtual functions.

C++ Code Example:
    #include 
    using namespace std;

    class Animal {
    public:
        virtual void makeSound() {
            cout << "Some generic animal sound" << endl;
        }
    };
    int main() {
        Animal* a1 = new Dog();
        Animal* a2 = new Cat();

        a1->makeSound();  // Calls Dog’s version
        a2->makeSound();  // Calls Cat’s version

        delete a1;
        delete a2;
        return 0;
    }
Output:

Barking...

Meowing...

Key points to remember:

Polymorphism - one interface, multiple implementations.

Achieved in C++ by virtual functions (dynamic binding).

Increases flexibility and reusability of code.
